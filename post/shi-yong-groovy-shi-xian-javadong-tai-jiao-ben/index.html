使用Groovy实现Java“动态”脚本

作者: Jobs.Lee's Blog 日期: 2021-07-25 阅读时间:2.0 分钟 字数:474 分类: Java 阅读量: -

基本背景

很多时候，我们线上项目需要执行一些数据初始化或者数据修复，然而在某些场景下，使用单纯的MySQL已经无法满足复杂业务上的一些操作；

为了降低错误和提高效率，考虑使用熟悉的Java语言写脚本。但是传统的Java语言是静态编译的语言，要么手动替换Class重启，要么用arthas进行热更新，都比较麻烦，有什么比较好的方式方法来支持Java的动态执行吗？当然还要兼容Spring环境下的Bean调用。

基本原理

Groovy是一个更加强大的JVM语言，本身支持脚本实现。

本项目其实就是基于Groovy语言脚本实现的一种Java"动态"脚本，支持Spring上下文的调用，本身使用其实也比较简单，给大家提供一种参考。

实现说明

需要引入的包

<!-- groovy -->
<dependency>
    <groupId>org.codehaus.groovy</groupId>
    <artifactId>groovy-all</artifactId>
    <version>3.0.8</version>
    <type>pom</type>
</dependency>

Groovy脚本的运行代码

Binding groovyBinding = new Binding();
groovyBinding.setVariable("param", param);
GroovyShell groovyShell = new GroovyShell(groovyBinding);
Script scriptObject = groovyShell.parse(script);
scriptObject.run();

基本groovy脚本文件示例

package org.jobsl.hotdog.groovy

import org.jobsl.hotdog.service.HotdogService
import org.jobsl.hotdog.util.SpringContextUtil

def hotdog = { param ->
    result = "success"
    println("Start Groovy Script...")
    HotdogService hotdogService = SpringContextUtil.getBean("hotdogService");
    println("SpringObject=" + hotdogService)
    result = hotdogService.hotdogNice(param)
    println("SpringResult=" + hotdogService)
    println("End Groovy Script.")
    return result
}

hotdog(param)

参数及返回结果说明：

  • 脚本内置名为param的参数，可以根据需要作为方法传参

  • 脚本默认内置了返回结果(String)result，默认值为"success"可以在脚本中接收返回值作为参考

  • Spring相关bean的调用内置的SpringContextUtil.getBean方法
