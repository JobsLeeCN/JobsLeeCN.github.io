Mac M1芯片arm64架构Java开发者临时解决方案

作者: Jobs.Lee's Blog 日期: 2021-05-30 阅读时间:0.9 分钟 字数:241 分类: M1 Mac 总结 阅读量: -

苹果新Mac采用arm64架构的M1芯片，对于开发者来说，普通软件有Rosetta进行处理可以满足普通的日常使用，但是对于开发软件如IDEA、Java和MySQL等底层大型软件来说，使用Rosetta的性能是不理想的。

下面聊一下在目前阶段开发者的“最佳”解决方案。

IDEA
本人亲自尝试，老版本并未提供基于ARM64架构新编译的版本，且老版本基于Rosetta的运行有一定卡顿，推荐使用支持M1芯片的最新版本

Java
推荐使用Zulu，Zulu是OpenJDK的一个版本，稳定、可靠；对于目前来说OpenJDK与Oracle Java的差别已经很小

Docker
官方已经适配，直接使用最新版本即可。

Docker-MySQL
MySQL在Docker中的官方镜像目前还没有ARM64版本，可以取分支mysql/mysql-server使用

Docker-Redis
官方镜像已经支持ARM64版本
